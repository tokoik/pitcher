## 2. ビルド方法

このリポジトリは CMake を利用してビルドします．`pitch-stereo.c` 1 ファイルから `pitcher` 実行ファイルを生成します．

### 2.1 Windows (Visual Studio 2022)

前提:

- Visual Studio 2022 (C++ によるデスクトップ開発ワークロード)
- CMake 3.15 以上
- インターネット接続（初回ビルド時に freeglut を FetchContent で取得するため）

手順 (PowerShell):

```powershell
cmake -S . -B build-vs2022 -G "Visual Studio 17 2022"
cmake --build build-vs2022 --config Release
```

実行ファイル:

- `build-vs2022/Release/pitcher.exe`

補足:

- CMake 設定により、`freeglut` は自動ダウンロード・ビルドされます。
- 実行に必要な DLL はビルド後コマンドで同一フォルダへコピーされます。

### 2.2 macOS (Xcode)

前提:

- Xcode 16 以上
- CMake 3.15 以上

手順 (Terminal):

```bash
cmake -S . -B build -G Xcode
cmake --build build --config Release
```

Xcode から実行する場合:

- `build/pitcher.xcodeproj` を開き、`Release` で `pitcher` ターゲットを実行

生成されるアプリケーションバンドルの例:

- `build/Release/pitcher.app`

補足:

- macOS では `OpenGL` と `GLUT` のフレームワークをリンクします。

### 2.3 Ubuntu Linux (Makefile)

前提:

- GCC または Clang
- CMake 3.15 以上
- `make`
- 開発用パッケージ (`freeglut`/`OpenGL`/`pkg-config`)

例 (Ubuntu):

```bash
sudo apt update
sudo apt install -y build-essential cmake pkg-config freeglut3-dev libgl1-mesa-dev
```

手順 (CMake で Unix Makefiles を生成):

```bash
cmake -S . -B build-make -G "Unix Makefiles"
cmake --build build-make -j
```

実行ファイル:

- `build-make/pitcher`

## 3. 使用方法

### 3.1 起動

ビルド後に生成された `pitcher` 実行ファイルを起動します．

- Windows: `build-vs2022/Release/pitcher.exe`
- macOS: `build/Release/pitcher.app`
- Ubuntu: `build-make/pitcher`

起動すると、地面・投球位置を示す立方体・飛来するボールが描画されます．

### 3.2 基本操作

- マウス左クリック:
  - クリック位置の方向に向けて、1 個のボールを放出します。
- マウス中クリック:
  - `glutIdleFunc(0)` を呼び、連続再描画を止めます（見た目上はアニメーション停止に近い動作）。
- マウス右クリック:
  - 1 回だけ再描画を要求します。
- スペースバー:
  - ランダム方向へボールを 1 個追加し、以後は画面外へ出たボールを補充する「連続放出量」を 1 段階増やします。押すたびに補充対象の個数が増えます（減少操作は未実装）。
- `o` キー:
  - 視差を広げます。奥行き感が強くなりますが、見え方の負担が増える場合があります。
- `c` キー:
  - 視差を狭めます。奥行き感は弱まりますが、見やすくなる場合があります。
- `Esc` / `q` キー:
  - 終了します。

### 3.3 見え方調整のポイント

- 視差が強すぎると、二重像や眼精疲労の原因になりやすくなります。
- まずは小さな視差で開始し、少しずつ `o` / `c` で調整してください。
- 長時間連続して使用しないでください（詳細は「1. 使用条件」を参照）。

## 4. アルゴリズムと処理手順

この節では `pitch-stereo.c` に基づき、描画処理とボール挙動の流れを説明します。

### 4.1 データ構造と初期化

- ボールは `struct balllist` の双方向リストで管理されます。
  - アクティブなボール: `start` から `stop` の間
  - 未使用ノード: `reserve`
- 視差量は `parallax` で管理し、初期値は `P` です。
- ステレオ方式はマクロ `STEREO` で切り替えます。
  - `NONE`: 通常描画
  - `QUADBUF`: 左右バッファ描画
  - `BARRIER`: 視差バリア向けライン分離描画（本デモの既定値）

### 4.2 ボール放出アルゴリズム (`shot`)

- `reserve` からノードを取り出して新しいボールを生成します。
- 放出方向は乱数 `u`, `v` から計算し、初速度を次で与えます。

$$
v_x = S\sin u\cos v,\quad
v_y = S\sin v,\quad
v_z = S\cos u\cos v
$$

ここで $S$ は `SPEED` です。

- 初期位置は `(PX, PY, PZ)` に固定されます。

### 4.3 フレーム更新 (`update`)

各フレームで、各ボールに対して以下を実行します。

1. 位置更新（時間刻み `TIMESCALE`）
2. 重力加速度 `G` の適用
3. 画面外に出たボールをリストから回収し、必要に応じて再放出
4. 地面付近 (`py < 0.1`) で簡易反射（反発係数 0.8）

離散時間更新の概念式は以下です。

$$
v_y \leftarrow v_y + G\Delta t,\quad
\mathbf{p} \leftarrow \mathbf{p} + \mathbf{v}\Delta t
$$

ここで $\Delta t$ は `TIMESCALE` です。

### 4.4 立体視描画 (`display`)

- 投影は `glFrustum` で左右目ごとに水平方向をシフトして作成します。
- 視差量 `parallax` から左右視点のずれを作り、奥行きを表現します。
- `BARRIER` モードではステンシルとカラーマスクを利用して、走査ラインごとに左右目画像を交互に合成します。
  - 奇数/偶数ラインをステンシルで分離
  - 右目・左目を色チャンネル単位で交互描画
- 最後に `glutSwapBuffers` で表示し、直後に `update` を実行して次フレームに進みます。

### 4.5 入力イベント処理

- `keyboard`:
  - `Esc` / `q`: 終了
  - `space`: ボール追加
  - `o` / `c`: 視差調整
- `mouse`:
  - クリック座標から方向ベクトルを作り、1 球を狙った方向に放出
- `resize`:
  - ビューポート更新
  - `BARRIER` モード時はステンシルにラインマスクを再構築
